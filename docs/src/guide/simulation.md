# Network Simulation

ERGM.jl provides tools to simulate networks from fitted ERGM models using Markov Chain Monte Carlo (MCMC). Simulated networks are essential for assessing model fit, making predictions, and understanding the structural implications of estimated parameters.

## Why Simulate?

Network simulation serves several purposes:

- **Goodness of fit**: Compare observed network properties to properties of networks generated by the model
- **Model exploration**: Understand what kinds of networks a given model specification produces
- **Prediction**: Generate plausible networks given estimated parameters
- **Hypothesis testing**: Use simulation-based tests for network hypotheses

## Simulating from a Fitted Model

The simplest approach is to simulate from an `ERGMResult`:

```julia
using Network, ERGM
using Random

Random.seed!(42)

# Fit a model
terms = [Edges(), Triangle(), NodeMatch(:gender)]
result = ergm(net, terms; method=:mple)

# Simulate 100 networks from the fitted model
sim_nets = simulate_ergm(result; n_sim=100)

println("Generated $(length(sim_nets)) networks")
println("Each with $(nv(sim_nets[1])) nodes")
```

### Parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| `n_sim` | Number of networks to simulate | 1 |
| `burnin` | MCMC burn-in steps | 10000 |
| `interval` | Steps between saved samples | 1000 |

### Choosing Burn-in and Interval

The burn-in ensures the MCMC chain has reached stationarity before sampling. The interval (thinning) reduces autocorrelation between samples.

| Network Size | Recommended Burn-in | Recommended Interval |
|-------------|--------------------|--------------------|
| Small (< 50 nodes) | 10,000 | 1,000 |
| Medium (50-200 nodes) | 50,000 | 5,000 |
| Large (> 200 nodes) | 100,000 | 10,000 |

```julia
# For a larger network
sim_nets = simulate_ergm(result;
    n_sim = 100,
    burnin = 50000,
    interval = 5000
)
```

## Simulating from a Model Specification

You can also simulate directly from an ERGM model and coefficient vector, without a fitted result:

```julia
# Define model
formula = ERGMFormula(terms)
model = ERGMModel(formula, net)

# Specify coefficients
θ = [-2.0, 0.5, 0.8]  # edges, triangle, nodematch

# Sample networks
sim_nets = sample_networks(model, θ; n_sim=50)
```

### Starting Network

By default, `sample_networks` initializes from a random network matching the observed density. You can provide a custom starting network:

```julia
# Start from the observed network
sim_nets = sample_networks(model, θ;
    n_sim = 50,
    start_net = net
)

# Start from an empty network
empty_net = Network{Int}(; n=nv(net), directed=is_directed(net))
sim_nets = sample_networks(model, θ;
    n_sim = 50,
    start_net = empty_net
)
```

## The MCMC Algorithm

Each simulation step:

1. **Propose**: Randomly select a dyad $(i,j)$
2. **Compute**: Calculate change statistics $\delta(y)_{ij}$ for all terms
3. **Accept/Reject**: Toggle the edge with probability:

$$\alpha = \min\left(1, \exp\left(\theta^\top \delta(y)_{ij}\right)\right)$$

If the edge currently exists, the acceptance probability uses the negative of the change statistics (removing vs. adding).

### Random Initialization

When no starting network is provided, a random network is created with edge probability matching the observed network density:

```julia
# The density of the observed network
density = ne(net) / (nv(net) * (nv(net) - 1) / 2)  # undirected
```

## Analyzing Simulated Networks

### Comparing Observed vs. Simulated Properties

```julia
# Edge count comparison
obs_edges = ne(net)
sim_edges = [ne(s) for s in sim_nets]

println("Observed edges: ", obs_edges)
println("Simulated edges: ",
    round(mean(sim_edges), digits=1), " ± ",
    round(std(sim_edges), digits=1))
```

### Checking Model Statistics

```julia
# Compute model statistics for each simulated network
ts = TermSet(terms)

for (i, snet) in enumerate(sim_nets[1:5])
    stats = compute_all(ts, snet)
    println("Simulation $i: ", round.(stats, digits=2))
end

# Compare to observed
obs_stats = compute_all(ts, net)
println("Observed:      ", round.(obs_stats, digits=2))
```

### Network-Level Summaries

```julia
using Statistics

# Density
sim_densities = [ne(s) / (nv(s) * (nv(s) - 1) / 2) for s in sim_nets]
println("Simulated density: ",
    round(mean(sim_densities), digits=4), " ± ",
    round(std(sim_densities), digits=4))

# Clustering coefficient (via Graphs.jl)
using Graphs
sim_clustering = [global_clustering_coefficient(s.graph) for s in sim_nets]
println("Simulated clustering: ",
    round(mean(sim_clustering), digits=4), " ± ",
    round(std(sim_clustering), digits=4))
```

## Exploring Parameter Effects

Use simulation to understand how parameters affect network structure:

```julia
# Vary the triangle coefficient
formula = ERGMFormula([Edges(), Triangle()])
model = ERGMModel(formula, net)

for tri_coef in [0.0, 0.5, 1.0, 1.5]
    θ = [-2.0, tri_coef]
    sims = sample_networks(model, θ; n_sim=20)
    avg_triangles = mean(compute(Triangle(), s) for s in sims)
    avg_edges = mean(ne(s) for s in sims)
    println("θ_triangle = $tri_coef → ",
        "edges: $(round(avg_edges, digits=1)), ",
        "triangles: $(round(avg_triangles, digits=1))")
end
```

## Computational Notes

- Simulation time scales with `burnin + n_sim × interval` MCMC steps
- Each step involves computing change statistics for all terms
- For large networks, consider reducing `n_sim` or increasing `interval`
- Simulated networks share the same node count and directedness as the observed network
- Only edge structure is simulated — vertex and edge attributes are not transferred to simulated networks

## Best Practices

1. **Set random seeds**: Use `Random.seed!()` for reproducible simulations
2. **Adequate burn-in**: Ensure the chain has reached stationarity
3. **Sufficient thinning**: Reduce autocorrelation between samples
4. **Compare statistics**: Always compare model statistics between observed and simulated networks
5. **Multiple simulations**: Use at least 50–100 simulated networks for stable estimates
6. **Check mixing**: If simulated networks are very similar, increase the interval
